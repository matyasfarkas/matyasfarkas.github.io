    Monetary-Financial Model — Documentation¶

    The Monetary-Financial Model is an interactive forecasting tool that runs entirely in your browser. It shows quarterly forecasts for about fifty economies — growth, inflation, interest rates, public finances, and more — and lets you bend them: click on a chart where you think inflation will be, and every other chart updates so the whole forecast stays consistent with the estimated economics. Thirty-three of the economies are linked through trade and financial channels, so pushing the United States moves the euro area, Japan, and China too.

    Three rules govern everything on the screen:

    1. Data are data. Solid history is observed. Where a country's latest quarter has not been published yet, the model fills the gap using everything already observed elsewhere — and those filled-in points are always drawn in amber, never disguised as data.
    2. The model is estimated once, then frozen. Within the year, new data releases shift the starting point of the forecast (the panel currently starts projections at 2026 Q1) but never silently change the model itself. Re-estimation happens once a year, on a complete final year of data. Each quarterly update is archived as a vintage, and the Monitor page shows exactly what each data release did to the forecasts.
    3. Judgment is visible. If you move a forecast away from what the model says — a scenario, a long-run target, a policy counterfactual — the chart says so.

    Under the hood it is a Bayesian Vector Autoregression (BVAR) per economy, coupled into a Global VAR (GVAR) with a fully endogenous United States, plus DSGE impulse responses for policy counterfactuals. The deployed tool is served at the /bvar-scenario path of the academic website, hosted on GitHub Pages (the matyasfarkas/starter-hugo-academic Hugo Academic site). The build/deploy chain is documented in Build & CI.

    Start here¶

    • User Manual (PDF) — the manual is in two parts. Part I, "Using the tool", is written for someone with no econometrics background: each section says what you can do, how to do it (step by step, naming the exact buttons), and what you will find on the screen. If you have five minutes, read the quick-start section and skip the rest until you need it. Part II, "How the machinery works", has the full methodology — BVAR estimation, COVID-as-missing data augmentation, the GVAR coupling and closed-form solve, the fiscal accounting, long-run targets, vintages, and reproducibility.
    • User Guide — a short web quick-start: your first scenario in five minutes, and where to go next. Defers to Part I of the manual for everything else.

    Empirical results are in

    An out-of-sample horse-race (56 origins, 2010Q1–2024Q4, × 9 models, CRPS-primary scoring + 90% Model Confidence Set) and the endogenous-US Tier-A spillover analysis have run. The headline: AR(p)/DFM lead the point race, but the EM-Bayesian GVAR is inside the Model Confidence Set on the nominal block (inflation at every horizon, the short rate at h ≤ 4), decisively beats the textbook OLS classical GVAR, and the endogenous-US variant weakly dominates the frozen anchor. Read the numbers and the honest caveats (over-confident draw densities; provisional spillover magnitudes) in Empirical Evaluation.

    Methodology and data¶

    • Fiscal Block Methodology — how the estimated fiscal drivers relate to the CBO and IMF fiscal-policy frameworks, the exact debt-dynamics recursion (snowball + SFA closure + primary balance), and the known limitations (including the China nominal-GDP inflation double-count).
    • Data & sources — per-source-family documentation (FRED / Eurostat / OECD-QNA / IMF-WEO), the 2025 estimation vintage versus the frozen-model quarterly refresh, and the full country × variable coverage matrix are in the User Manual (PDF) (Part II, §Data Sources), generated from app/public/data/coverage_manifest.json. Where a series lags its country's release calendar, the gap to the common 2026 Q1 origin is filled by the cross-country model and flagged as a nowcast in the manifest (imputedThrough) — the same cells the app draws in amber.
    • Empirical Evaluation — the out-of-sample forecast horse-race (9-model density race, CRPS + Model Confidence Set), the endogenous-US Tier-A spillover analysis, and the honest caveats. Implemented in a separate evaluation repository.
    • Paper Outline (PDF) — the working outline of the accompanying paper.

    For developers and maintainers¶

    • Architecture (for AI / developers) — dense codebase map of the compute engine, GVAR topology, fiscal block, data conventions, and tests. Also documents the drag-perf / UX architecture (§5a/§8b): the heavy ~4.5 s GVAR solve runs off the live drag path — zero solves during a drag, exactly one cheap "direct" solve on pointer-up, and the full cross-country fixed point with uncertainty bands only on the Solve full GVAR button; the optimistic conditioned dot (conditionedDotY.ts) snaps to the click instantly; and per-chart React.memo repaints one panel, not all ~35.
    • Pipeline — the offline build → estimate → check → port → couple → test → deploy pipeline that produces the shipped data.
    • Modules & API — module-level reference for the compute and data layers.
    • Data Schemas — the JSON contracts the browser app reads.
    • Build & CI — the green gate, the test taxonomy, and the deploy chain.
    • Maintainer's Guide — operations, adding a country, and re-estimating models. Quarterly vintage shifts run through app/scripts/auto_update.mjs (--check, --shift --asof), which archives the old vintage, imputes ragged edges cross-block, rebuilds provenance, and writes the release-impact report behind the /monitor page.
