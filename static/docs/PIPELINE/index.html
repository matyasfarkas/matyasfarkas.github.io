    Data & Estimation Pipeline¶

    This page documents the offline pipeline that turns raw public data into the static JSON the web app serves: build → estimate → check → port → couple → test → deploy. Everything here runs on the researcher's machine (MATLAB + Python + Node); the deployed browser app only ever reads the resulting JSON.

    Scope / where to look next. This page is the mechanism reference — the script inventory, the raw-data precedence chain, the GVAR-EM loop internals, the prior/coherence tagging sweeps, and the port/coupling steps. For the decision tree ("what kind of update is this, and what does it re-estimate?") and the operational runbooks (exact copy-paste sequences with the green gate) see MAINTAINER_OPERATIONS.md, REESTIMATING_MODELS.md and ADDING_A_COUNTRY.md; for the live compute architecture see ARCHITECTURE_FOR_AI.md; for the fiscal block see fiscal-block-methodology.md. The per-economy variable table (transformation · prior · financial-tag, all 1048 ported variable rows audited coherent) lives in the user manual (docs/user_manual.tex, sec:varcoherence). The out-of-sample forecast horse-race + endogenous-US spillover evaluation lives in a separate repo — see §10 and its docs/EMPIRICAL_PROTOCOL.md. This page does not repeat those — it complements them.

    The deployed platform is ~50 economies: 33 coupled GVAR blocs (the US dominant unit + 32 partners, with the euro area as one aggregate EA bloc) + 17 euro-area FSAP standalones (the euro members, estimated in separation). Data is harmonised to a common 2025Q4 vintage from FRED, Eurostat, BIS, ECB, OECD and IMF sources.

    3.3 The fiscal stationary ("WN") prior — tag_fiscal_wn.py¶

    The GLP/Minnesota prior in BVAR/MFBVAR/GLPreplicationWeb/bvarGLPmf.m builds the prior coefficient matrix b with a unit own-first-lag mean by default (a random walk: diagb=ones(n,1)). Any column listed in the pos vector gets its own-first-lag prior mean zeroed, turning that variable into a mean-reverting white-noise (WN) process under the prior:

    % BVAR/MFBVAR/GLPreplicationWeb/bvarGLPmf.m, lines 192–195
b=zeros(k,n);
diagb=ones(n,1);
diagb(pos)=0;          % zero the own-first-lag prior mean for the 'pos' columns -> stationary/WN
b(2:n+1,:)=diag(diagb);

    pos is the 3rd positional argument of the COVID-as-missing sampler cgl_bvarV5 (which renames it pos=stationary; and forwards it as 'pos' to bvarGLPmf):

    % BVAR/MFBVAR/cgl_bvarV5.m
function [Xcond,storeP,BETA,SU,X_interp0,Dforecast] = cgl_bvarV5(x,p,stationary,nDraws,...)
pos=stationary;
... r = bvarGLPmf(..., 'pos', pos);

    Why this matters. Before this change every call passed [] for stationary, so every variable in every bloc was a prior random walk. The fiscal GDP ratios — FREV_GDP (revenue/GDP), FPEXP_GDP (primary expenditure/GDP), and SFA (the stock-flow adjustment) — are economically stationary: they fluctuate around a long-run share, they do not drift. Under the RW prior the projected ratios stayed pinned near their last observation at the long horizon (US revenue stuck near ~19%). The WN prior lets them mean-revert to their in-sample means — US FREV_GDP → ~17.9% (sample mean 17.91, last obs 18.77; commit e38673a) instead of drifting. The effective interest rate EFFI is genuinely persistent and stays RW.

    How the tag is consumed. Each estimator reads the per-variable prior field out of the raw JSON via a priorsfrom helper and builds the WN column-index vector, passed as cgl_bvarV5's 3rd arg:

    % export_bvar_gvar24_loop.m — on the kept domestic block, indices into x=[dom | stars]
statCols = find(Spec.Prior(keep)=="WN")';   % empty for the 44 non-fiscal blocs
[Xcond,~,B,S,~,Dfc] = cgl_bvarV5(x, lags, Dc.statCols, nDraws, nBurn, ...);

% export_bvar_euro_fsap.m
statCols = find(Prior=="WN")';
[~,storeP,BETA,SU,~,Dforecast] = cgl_bvarV5(x, lags, statCols, nDraws, nBurn, ...);

    Because the stars (ROW*) are appended after the domestic columns, the WN indices (which point into the domestic block) are unchanged by the star append — and they remain on the domestic ratios only; stars are always RW. statCols is empty for the 44 non-fiscal blocs, where pos=zeros(1,0) behaves exactly like the old [] — so the WN change is a byte-identical no-op for every bloc that carries no fiscal ratio.

    priorsfrom and the cell-array trap. tag_fiscal_wn.py writes prior on every variable object (not just the 3 WN ones) so the field set stays homogeneous: MATLAB's jsondecode returns J.variables as a struct array only when all objects share the same fields; a partial prior tag makes it a cell array and breaks the estimators' arrayfun(@(v)v.id, J.variables) calls. priorsfrom (present in both estimators) is written to be robust to either shape. The Python tagger is format-preserving (a targeted text splice — only the inserted/updated prior bytes change; the data array is untouched) and idempotent (re-running once tags are present is a no-op). It edits the globals-augmented file the GVAR loop actually reads, preferring _glob → _2025q4 → plain.

    python3 pipeline/tag_fiscal_wn.py
# DataRaw_US_glob.json: 0 field(s) changed this run; WN = ['FPEXP_GDP', 'FREV_GDP', 'SFA']; RW count = 35
# ... (idempotent: 0 changes once tagged)

    The exported spec.json carries the real prior per variable (WN/RW), so the contract and the per-economy coherence table reflect it. Committed in 6f0161a (plumbing) and 22250be/e38673a (the refit). See the fiscal recursion in fiscal-block-methodology.md.

    3.4 Global-commons financial coherence — tag_global_financial.py¶

    The COVID-as-missing masking in the estimator NaN's the 2020Q2–Q4 window only for the slow national-accounts macro variables (isFinancial:0), so the Durbin–Koopman smoother imputes them; fast-moving market prices (isFinancial:1) are kept observed through COVID and condition that imputation (§5.1, step 3). An audit found two of the four US-determined global commons mis-tagged: GLB_USEQ (US equity index) was isFinancial:false in every non-US bloc, and GLB_OIL (oil price) was isFinancial:false in every bloc — so they were being NaN-imputed across COVID like a GDP series, which is incoherent for a daily-quoted market price. (GLB_USLR long rate and GLB_USBAA BAA spread were already financial.)

    tag_global_financial.py flips both to isFinancial:1 in every DataRaw_<CC>_glob.json so they match WTI_US/PZTEXP and stay observed through COVID. It is a value-only splice (the isFinancial field already exists on every object, so it never inserts a field — avoiding the same cell-array trap as §3.3), format-preserving, and idempotent. The source-of-truth fix is in build_globals_new_blocs.py, which now forces all four globals (GLB_FINANCIAL = {GLB_OIL, GLB_USLR, GLB_USBAA, GLB_USEQ}, line 37) financial when a new bloc is built, so future rebuilds stay coherent; the tagger is the one-time sweep over already-built globs.

    python3 pipeline/tag_global_financial.py
# DataRaw_CA_glob.json: 0 field(s) changed this run; {'GLB_USEQ': 1, 'GLB_OIL': 1}

    Committed in 22250be/e38673a.

    3.5 Canada IP as a log-level — build_ca_ip_level.py¶

    The CA glob's IP_CA column had been stored as a y/y growth rate (transformation:"lin"), incoherent with the US/UK IP blocs (stored as levels, transformation:"log"). build_ca_ip_level.py rebuilds it as a quarterly level index using the same growth-chaining convention as build_gvar_2025q4.py (level_t = level_{t-1}·(1+g_t)):

    1. average the discontinued monthly FRED level index CANPROINDMISMEI (Index 2015=100, ends 2024-02) to complete quarters (last complete quarter ≈ 2023-Q4);
    2. chain forward from there with the q/q growth series PRINTO01CAQ657S (to 2025-Q4): level_t = level_{t-1}·(1 + g_t/100);
    3. align onto the CA glob's exact quarterly grid (1998-Q1 … 2025-Q4), replace the IP_CA column, and flip the spec to transformation:"log", isFinancial:0 (prior RW unchanged).

    It reads FRED via the public fredgraph.csv endpoint (no API key), caching CSVs under pipeline/country_raw/. It asserts the reconstructed level is positive/finite/smooth (no q/q spike beyond ~±20%) and ends near the observed ~106–107 range.

    python3 pipeline/build_ca_ip_level.py
# WROTE .../DataRaw_CA_glob.json (IP_CA now LEVEL, transformation=log, isFinancial=0)

    Committed in 9839a46 (the CA-only targeted re-estimate, §10.1) and folded into the full refit e38673a. After this rebuild, CA's IP propagates correctly through partners' ROW* stars (the earlier CA-IP approximation is removed).

    Coherence audit. All 1048 ported variable rows (across the 50 blocs in app/public/data/bvar/<CC>/spec.json) were audited for transformation·prior·isFinancial coherence; the per-economy table is in the user manual (sec:varcoherence, commit fa104cd).

    4. The DataRaw precedence chain¶

    Each bloc has up to three raw-input layers; the GVAR-EM loop picks the first that exists (export_bvar_gvar24_loop.m, the per-bloc load loop):

    raw = DataRaw_<CC>_glob.json        % globals-augmented (domestic + 4 GLB_* commons; _US dropped)
if missing → DataRaw_<CC>_2025q4.json   % common 2025Q4 endpoint (WEO-Denton extended)
if missing → DataRaw_<CC>.json          % the original FRED/Eurostat pull
    Layer Built by Contents
    DataRaw_<CC>_glob.json build_globals_new_blocs.py (+ fiscal/financial editors) domestic block (legacy _US removed) + 4 global commons; the fiscal columns live here
    DataRaw_<CC>_2025q4.json build_gvar_2025q4.py domestic series extended to a common 2025Q4 endpoint (balanced panel)
    DataRaw_<CC>.json build_country_data.py / build_oecd_*.py the original pull (domestic + US block)

    The _glob file is the production input: it carries the 4 globals the GVAR estimator expects, and it is the file the 4-variable fiscal builders edit in place (the SFA column). The 3 RoW stars are not stored in the glob — the loop computes them at solve time from partners' completed panels and appends them only at export.

    Important. This precedence is offline only — it governs which raw input the MATLAB estimator reads. The browser app never sees DataRaw_*; it reads the ported app/public/data/bvar/<CC>/ JSON.

    5. The GVAR-EM loop (export_bvar_gvar24_loop.m)¶

    The loop is an EM-style fixed-point over the cross-country foreign components. It interleaves a Gibbs/DK imputation step (estimate each bloc, fill the COVID + foreign-star holes) with a trade-weighting step (rebuild every bloc's RoW stars from partners' completed panels), and iterates until the foreign components stop moving.

    5.1 Per-iteration internals¶

    1. Bootstrap the US (closed anchor). Estimate the US once with no stars (cgl_bvarV5(xUS, …)) over the COVID-masked domestic block, take the posterior-mean DK-completed series XcUS = mean(Xcond,3) and form its [realGDP, price, rate] panel. This seeds iteration-1's stars. With US_ENDOGENOUS=1 the US is then folded into the loop and re-estimated with its own ROW*_US stars (the no-stars fit is discarded — a warm start, not exported).
    2. Build stars (buildstars). For each bloc, ROWDEM/ROWINF/ROWFIN = trade-weighted averages of partners' real-GDP growth / inflation / short rate, renormalised over the partners present each quarter (unbalanced panel). Stars are 4·Δ(100·log) (annualised %), not 400·. Per-component availability: a rate-less partner (CH/NO) still feeds DEM/INF; hyperinflation partners (AR) are dropped from INF/FIN but kept in DEM. Leading NaN is back-filled; internal COVID NaN is left for the smoother.
    3. Estimate each bloc (parfor). x = [domestic | ROWDEM ROWINF ROWFIN]; over 2020Q2–Q4 NaN only the macro columns (domestic isFinancial==false + ROWDEM + ROWINF); keep all financial columns + ROWFIN observed. The financial set kept observed now includes all four global commons GLB_OIL/GLB_USLR/GLB_USBAA/GLB_USEQ (after the §3.4 coherence fix), matching the domestic market prices (WTI_US/PZTEXP). The WN/stationary fiscal columns enter as the 3rd positional statCols arg (§3.3). cgl_bvarV5(..., statCols, ..., 'sur',0,'noc',1,'Ndraws',1000, 'Ndrawsdiscard',500) then imputes the macro holes via the Durbin–Koopman smoother conditioned on the observed financials. The completed panel is the posterior mean mean(Xcond,3).
    4. Converge. Compute max|Δstar| across blocs (excluding the COVID-NaN'd macro-star cells, which are MC noise; ROWFIN stays in the metric). Stop when it>1 && max|Δstar| < TOL.

    This is a Jacobi sweep: all stars are built from the frozen start-of-iteration state, then all non-US blocs (or all blocs when US_ENDOGENOUS) are estimated independently in parallel — same fixed point as Gauss–Seidel, ~Ncores faster.

    Algorithm (EM-style fixed point over the foreign components).

    Input:  per-bloc panels {X_c}, trade weights W, COVID window 2020Q2..Q4, TOL, MAXIT
Output: per-bloc completed panels {X̃_c} + posterior draws (the 8-file contract)

  ── E-step seed ───────────────────────────────────────────────
  Estimate US with NO stars (cgl_bvarV5, COVID-macro-masked) → X̃_US⁰ = mean(Xcond,3)
  Seed each bloc's foreign stars S_c⁰ from the iteration-0 completed panels.

  ── EM loop (Jacobi) ──────────────────────────────────────────
  for it = 1 .. MAXIT:
    M-step / star build:   for every bloc c:               # frozen start-of-iter state
        S_c = buildstars(W, {X̃_p}_{p≠c})                   # ROWDEM/ROWINF/ROWFIN, 4·Δ(100·log),
                                                            #   per-quarter renormalised over present partners
    E-step / re-estimate:  parfor every bloc c:
        x_c = [domestic_c | S_c]                            # WN cols = statCols (3rd arg)
        NaN the 2020Q2..Q4 MACRO cells (isFinancial==false + ROWDEM + ROWINF) only;
            keep all financials + ROWFIN (incl. GLB_*) observed
        [Xcond, β_c, Σ_c, ...] = cgl_bvarV5(x_c, lags=3, statCols, nDraws, nBurn, 1, H, ...)
        X̃_c = mean(Xcond,3)                                 # DK-smoother posterior-mean completion
    convergence:  Δ = max_c |S_c^{it} − S_c^{it−1}|         # exclude COVID-NaN'd macro-star cells
    if it>1 and Δ < TOL: break                              # MC noise floor ≈ 0.13

  save /tmp/gvar24_workspace.mat   then   exportbloc(c) → /tmp/gvar24_export/<c>/ (8 files)

    The E-step is the Durbin–Koopman data-augmentation completion of the foreign blocs under COVID-macro-masked / financials-observed missingness; the M-step is the Bayesian (GLP) re-estimation plus the trade-weighted star rebuild. Iterating to a fixed point in the completed stars is what couples the blocs at estimation time (distinct from the solve-time coupling x=Mx+d in §7).

    5.3 The 8-file per-bloc contract¶

    Each export dir contains:

    File Contents
    spec.json { n, variables:[{id, name, transformation('log'\|'lin'), isFinancial, prior, block}] } — block ∈ domestic/external/us/global/row/fiscal; prior ∈ RW (own-first-lag mean 1, default) / WN (own-first-lag mean 0, stationary — the 3 fiscal ratios FREV_GDP/FPEXP_GDP/SFA, §3.3)
    companion_mean.json { betahat, sigmahat, n, lags }
    companion_draws.json { beta, beta_shape, sigma, sigma_shape, n_draws, n, lags } (posterior draws → Full uncertainty bands)
    baseline.json unconditional forecast in growth/display space (mean, q05, q16, q84, q95, horizon)
    baseline_levels.json same forecast in level space
    historical.json / historical_levels.json last ~40 quarters (growth / level) — observed data, not the DK counterfactual
    transformations.json per-variable 'log'/'lin'

    Plus, where applicable: fiscal_meta.json { d0, sfaDefault, idx:{gdp,deflator,revenue,primaryExp,effRate,sfa} } (copied from pipeline/fiscal_meta/<CC>.json) and the GVAR weights_*.json.

    🔴 port_gvar_exports.py does NOT regenerate fiscal_meta.json's positional indices. The 33-bloc refit inserts variables ahead of the fiscal block in every spec except US, so a verbatim port leaves the six idx offsets pointing at the wrong columns — the EA-effRate-lands-on-equity (~col 537) debt explosion. node pipeline/fix_fiscal_meta_indices.mjs re-resolves all six BY VARIABLE ID (FREV_GDP/FPEXP_GDP/EFFI/SFA; gdp=0; deflator=first HICP|CPI|JGDP|PGDP|DEFL) from the deployed spec.json, preserves d0/sfaDefault, and is idempotent. The fiscal-meta-indices.test.ts canary blocks a deploy whose indices are stale (every fiscal bloc's derived debt path must stay bounded).

    7. Coupling (precompute_gvar_coupling.mjs) — the pre-shipped inverse¶

    The browser solves the cross-country fixed point x = (I−M)⁻¹ d from a pre-shipped coupling artifact. The in-browser M build (DK-smoother probes) is ~19 min at 24 blocs and worse at the augmented endogenous dimension, so pre-shipping is mandatory, not an optimisation.

    app/scripts/precompute_gvar_coupling.mjs (run via npx vite-node):

    1. assembles the deployed bloc bundle exactly like the loader (a non-US bloc qualifies iff its spec.json has block:'row' vars; US is always included), self-configures the topology, and computes modelSig;
    2. builds M in JS (buildGVARCouplingM / buildGVARCouplingMEndo when US_ENDOGENOUS);
    3. inverts (I−M) via MATLAB BLAS (app/scripts/invert_coupling.m; the NumPy invert_coupling.py is the no-MATLAB fallback);
    4. writes the gzipped app/public/data/bvar/gvar/coupling_default.json.gz ({ modelSig, pinStructure:'', dim, rhoSpectral, M, invImM }, 6-sig-fig). The plain .json is removed (and the loader still falls back to a plain .json if no .gz exists). At the current 33-bloc dim 2060 the artifact is ~31 MB gzip (the live file is 32,166,056 B) vs ~91 MB raw — already most of GitHub's 100 MB per-file limit, and the script projects ~200 MB raw at the full ~50-bloc panel, so gzip is mandatory. The loader decompresses it in-browser via the native DecompressionStream.
    cd app
npx vite-node scripts/precompute_gvar_coupling.mjs

    Keyed to the data by modelSig. The artifact MUST be regenerated whenever any deployed bloc changes — a stale modelSig is rejected and the app falls back to the slow path.

    The augmented operator's spectral radius grows with the bloc count (it must stay < 1 for a stable fixed point):

    Topology dim ρ(M)
    6-bloc — 0.59
    24-bloc (frozen anchor) — 0.726
    33-bloc (endogenous US) 2060 0.924041 (live coupling_default.json.gz)

    The exact live rhoSpectral moves slightly with each refit (the shipped artifact reads 0.924041 at dim 2060; earlier artifacts read ~0.89). What matters is that it stays comfortably < 1. The artifact's modelSig (per-bloc n_lags_H_<betahat-fingerprint>) keys it to the exact data; US shows 41 confirming the endogenous-US augmentation.

    10. Targeted single-bloc re-estimation¶

    Two distinct "re-estimate one bloc" patterns exist; picking the wrong one is the most common maintenance mistake. The rule is set by whether the bloc is in the coupled GVAR. See the decision tree in MAINTAINER_OPERATIONS.md §0; this is the mechanism behind it.

    The bloc is… Pattern Re-estimates Coupling regen?
    a euro-area FSAP standalone (AT BE DE EE ES FI FR GR IE IT LT LU LV NL PT SI SK) truly single-bloc — it is not in the coupled system (no weights row, no block:'row' vars) only that bloc no
    a coupled GVAR bloc (US/EA/any RoW) full GVAR re-estimate — its stars feed every other bloc, so the fixed point moves for everyone all 33 yes

    10.1 The euro standalone — genuinely isolated¶

    A euro member is a self-contained FSAP two-country BVAR (domestic + exogenous US block + 4 globals), estimated by export_bvar_euro_fsap.m. It carries no coupling, so re-estimating it touches nothing else:

    # build/edit its DataRaw, then estimate just that member (subset via BVAR_EURO_MEMBERS)
BVAR_EURO_MEMBERS="DE" matlab -batch "run('pipeline/export_bvar_euro_fsap.m')"   # → /tmp/euro_fsap_export/DE/
GVAR_EXPORT_DIR=/tmp/euro_fsap_export GVAR_PORT_BLOCS="DE" python3 pipeline/port_gvar_exports.py
# NO coupling regen (not in the GVAR); then the green gate (§8)

    10.2 The coupled bloc — there is no "just one bloc" path¶

    Because each coupled bloc's ROW* stars are trade-weighted aggregates of every partner's completed panel, changing one coupled bloc's data perturbs the EM fixed point for all of them. The honest path is the full loop. But you can iterate fast on a subset using BVAR_GVAR_BLOCS (US first), which is the validate-then-commit pattern: confirm a data/coherence fix is correct on a 2–3-bloc smoke run before launching the multi-hour full refit. This is exactly how the §3.5 CA-IP fix and the §3.3/§3.4 tags were landed:

    # (a) FAST validation subset (separated, no coupling) — confirm the fix is sane
BVAR_GVAR_BLOCS="US CA" NDRAWS=200 MAXIT=1 US_ENDOGENOUS=1 \
  matlab -batch "run('pipeline/export_bvar_gvar24_loop.m')"
GVAR_PORT_BLOCS="US CA" python3 pipeline/port_gvar_exports.py   # inspect /tmp/gvar24_export first

# (b) once happy, the FULL coupled refit (the only deploy-grade path for a coupled change)
US_ENDOGENOUS=1 matlab -batch "run('pipeline/export_bvar_gvar24_loop.m')"
python3 pipeline/port_gvar_exports.py
cd app && npx vite-node scripts/precompute_gvar_coupling.mjs    # MANDATORY: stars changed for everyone

    The intermediate commits 6f0161a (WN plumbing) → 9839a46 (CA-only targeted re-estimate, the validate step) → 22250be → e38673a (the self-consistent full 33-bloc refit folding in all three fixes) are this pattern in the git history. A MAXIT=1 subset run is not deploy-grade for a coupled bloc — it freezes the stars at their historical (uncoupled) values; only the full MAXIT=12 loop reaches the cross-country fixed point. After any coupled change you must regenerate coupling_default.json.gz (§7) — a stale modelSig is rejected and the app silently falls back to the slow path. See MAINTAINER_OPERATIONS.md §C–§D for the per-bloc checks (n == spec == companion, convergence vs the MC floor, fiscal closure) and §E for the green gate.

    11. Empirical evaluation (out-of-sample horse-race + spillover)¶

    A separate repo — em-bayesian-gvar-eval (docs/EMPIRICAL_PROTOCOL.md) — evaluates the platform's forecasts against benchmarks. It consumes the same estimator and conventions documented here; this section is the cross-link, not a duplicate.

    Design (final). Two cores:

    1. Out-of-sample forecast horse-race. Recursive expanding-window, origins 2010Q1–2024Q4 (drop the 2020 COVID origins for the headline), targets {real GDP growth, inflation, short rate} for US/EA/JP/UK/CA/CN (full-panel = all 33 blocs in the appendix), horizons h ∈ {1, 2, 4, 8}.
    2. Proposed: the EM-Bayesian GVAR — endogenous-US (emgvar_endo, US_ENDOGENOUS=1) and the legacy frozen-anchor (emgvar_frozen).
    3. Benchmarks: Random Walk (driftless + a sample-mean "climatology" anchor), AR(p) (AIC-selected, iterated + direct), single-country Bayesian VAR (the coupling ablation = strip the ROW* columns, MAXIT=1), textbook OLS Pesaran–Schuermann–Weiner / Dees GVAR (cgvar_a), single-pass Bayesian GVAR (cgvar_b, MAXIT=1 — the EM ablation), and a Bayesian DFM.
    4. Leak-free by construction. Panels truncated ≤ t0 before transformData and before any star is built; rolling as-of-t0 trade weights (IMTS bilateral trade with endPeriod ≤ year(t0)) — the shipped 2021–2023 weights_full.json is forbidden in the sweep (it would inject future trade structure). COVID-as-missing is reapplied in-sample at every origin where 2020Q2–Q4 ≤ t0.

    5. Scoring: CRPS (primary density metric) + parametric log-score + PIT/coverage, scored from the per-draw predictive cloud Dforecast(1:8,:,:). Significance via Giacomini–White (headline — valid under nesting and correct for the expanding scheme), Clark–West (nested point), Diebold–Mariano (non-nested), with Romano–Wolf + the Model Confidence Set for multiplicity.

    6. Endogenous-US spillover analysis. The contribution the endogenous-US GVAR enables and the frozen-anchor cannot: a US shock that propagates abroad and feeds back into the US. Tier A (directly solver-computable conditional scenario propagation via solveGVARClosedFormEndo / solveGVARLinear in app/src/lib/compute/gvarSolve.ts) ships first; Tier B (GIRF/FEVD/connectedness) is a new module built from the exported companion_mean.json (betahat=Γ, sigmahat=Σ) + the coupling M — not a call into the current solver (which computes conditional-mean deltas, not GIRFs).

    Headline result (first full pass — results/tables.{txt,json}, 56 origins, 9 models; DESIGN final). AR(p)/DFM have the lowest point CRPS, but the 90% Model Confidence Set carries the value-add: emgvar_endo is in the inflation MCS at all horizons and the short-rate MCS at h=1,2,4 (out of the growth MCS at h=1 & h=8), emgvar_endo ⪰ emgvar_frozen, and it decisively beats the textbook OLS classical GVAR (cgvar_a removed from most sets). Best inflation h=1 = emgvar_endo (relRMSE 0.799). Honest caveats: the draw-based densities are over-confident (PIT cov90 ~0.73–0.83 vs 0.90) and the parametric log-score column is numerically broken (CRPS + MCS are the robust primaries). The Tier-A US +100 bp result shows non-trivial endogenous-only feedback the frozen anchor omits, but the magnitudes are large/sign-flipping and it is a multi-period conditional projection, NOT a causal IRF — validate before headlining. Do not cite the log-score numbers.

    See em-bayesian-gvar-eval/docs/EMPIRICAL_PROTOCOL.md for the full leak-free driver design (oos_driver.m), the per-origin forecast JSON schema, the compute plan, and the look-ahead invariants.
