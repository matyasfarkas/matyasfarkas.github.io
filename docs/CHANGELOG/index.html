    CHANGELOG¶

    [2.2.0] - 2026-06-16¶

    Drag-off-the-heavy-path GVAR UX (optimistic pin + right-click-to-plant + Compute-gated full solve), a consistent re-estimation lifting the coupling spectral radius to ρ(M) = 0.924, the fix_fiscal_meta_indices.mjs re-index guard for the EA/UK debt explosion, the produced out-of-sample horse-race results, and a documented known-issues list.

    Added¶

    • pipeline/fix_fiscal_meta_indices.mjs — fiscal-meta re-index (and a MANDATORY runbook RULE). fiscal_meta.json stores the fiscal-driver columns by position (idx = {gdp, deflator, revenue, primaryExp, effRate, sfa}); the 33-bloc GVAR-EM refit inserted variables ahead of the fiscal block in every spec except US, which silently re-pointed those positional indices at the wrong columns — EA's effRate landed on the equity index (~col 537) and the EA/UK debt-to-GDP path exploded. The new script re-resolves the indices by variable id from the deployed spec.json (order-independent) and rewrites all six fiscal metas, preserving d0/sfaDefault (it only touches meta.idx); it is idempotent. RULE (now documented in the maintainer runbooks): run node pipeline/fix_fiscal_meta_indices.mjs after ANY re-estimation/port that can change a fiscal bloc's variable order or count. The porter does not regenerate the meta, so this re-index is the fix — a green tsc/build with stale indices still ships a wrong debt path.
    • Optimistic conditioned-point pin (app/src/lib/conditionedDotY.ts). When you click, drag, or right-click to set a conditioned point, the marker (dot) now snaps to the clicked value instantly — it is reconstructed from the conditioning path (conditionedDotDisplayY), the exact inverse of the display→stored mapping (conditioningDeviationFromDisplayY), so it never waits for the (multi-second, on GVAR) forecast solve. The smoothed line + uncertainty bands reconcile around the pinned point once the solve lands. A single source of truth for the display↔stored round-trip prevents the two from drifting.
    • Right-click to plant a pin at the cursor. Right-clicking a forecast chart conditions that variable (if needed) and plants the conditioned point exactly at the cursor (the same forward mapping handleDrag uses, via onContextMenu in ChartGrid.tsx); left-clicking the "condition" button still toggles conditioning without moving a point.
    • Produced OOS empirical-evaluation results (separate repo em-bayesian-gvar-eval, results/tables.{txt,json} + results/tierA/us_rate_100bp.json). The recursive expanding-window horse-race — 56 origins (2010Q1–2024Q4, 2020 dropped) × 9 models, rolling as-of-t0 trade weights, CRPS-primary density score + 90% Model Confidence Set — now has numbers: AR(p)/DFM have the lowest point CRPS, but the EM-GVAR is in the inflation MCS at all horizons (h=1,2,4,8) and the short-rate MCS at h=1,2,4, statistically indistinguishable from the best on the nominal block, and decisively beats the textbook OLS classical GVAR (cgvar_a dropped from most MCS sets); emgvar_endo ⪰ emgvar_frozen in the MCS (best INFL h=1 relRMSE = emgvar_endo, 0.799). Honest caveats documented: the draw-based densities are over-confident (PIT cov90 ≈ 0.73–0.85 vs 0.90), the parametric log-score column is numerically broken on the under-covered clouds (CRPS + MCS are the robust primaries), and the Tier-A US-+100bp spillover is a multi-period conditional projection (large, sign-flipping magnitudes), not a causal IRF.
    • Known-issues / limitations sections in README.md and docs/MAINTAINER_OPERATIONS.md — five honestly-flagged, not-yet-fixed defects: (1) the CN fiscal debt inflation double-count (the snowball reads GDP_CN = nominal GDP plus a CPI deflator → inflation counted twice); (2) longRunTilt.ts does not filter non-finite long-run moments before the entropic-weight solve (a NaN draw can silently empty the anchor band); (3) the closed-form GVAR solve has no ρ(M)<1 divergence guard; (4) the conditional band uses the filtered (not DK-smoothed) covariance, overstating the band before a future pin; (5) the porter does not regenerate fiscal_meta (the stale-index cause of the EA/UK debt explosion — now guarded by the re-index RULE).

    Changed¶

    • Drag perf — the heavy GVAR solve is OFF the live drag path. On the live ~50-bloc US-endogenous panel every mean solve is multi-second (~4.5 s warm / ~13 s cold — the US-pinned structure misses the empty-pinStructure shipped coupling artifact), so the engine now withholds every solve during a drag (value-only or structure-change). The drag path is a cheap conditioning preview (the optimistic dot snaps to the cursor); exactly one cheap direct first-order solve fires on pointer-up; and the full cross-country fixed point + uncertainty bands run only on the "Solve full GVAR" button / top-bar Compute. A direct/full mode pill and a "computing…" badge make the mode explicit; the ≈95 MB coupling artifact is stripped from the worker post in direct mode. Drag state is a module-level signal bus (app/src/lib/compute/gvarDragState.ts) so begin/end/in-flight flags do not re-render the whole grid. Per-chart React.memo + reconciled chart references mean a single conditioning edit repaints one panel of ~35, not all of them.
    • Coupling spectral radius is now ρ(M) = 0.924 (was 0.896 / 0.896156 in an earlier refit). A later consistent re-estimation produced a new shipped coupling_default.json.gz (dim 2060, gzipped ≈ 31 MB) with a different modelSig and rhoSpectral = 0.924041 (still < 1, stationary; verified off the artifact header). Docs corrected to read the live value off the header rather than a copied number.
    • Docs accuracy pass. Corrected ρ(M) (0.896 → 0.924) across README.md, docs/ADDING_A_COUNTRY.md, docs/REESTIMATING_MODELS.md, docs/MAINTAINER_OPERATIONS.md (incl. the verify-command expect-line); fixed the fiscal bloc variable counts in README.md (EA 24, JP 23, UK 25, CA 25, CN 20 — previously conflated with the SFA estimation-sample lengths) and the stale GVAR-bloc count in USER_GUIDE.md (24 → 33, US now endogenous); replaced the stale ~0.45 s / ~3.4 s drag timings with the verified drag-off-the-heavy-path behaviour; documented the npm test (fast gate) vs npm run test:heavy split; and rewrote the generic app/README.md into a project-specific front-end guide.

    [2.1.0] - 2026-06-15¶

    Fiscal stationary prior + mean-reversion, GVAR coherence fixes, a full consistent 33-bloc refit, a per-economy variable table in the manual, and the out-of-sample empirical-evaluation harness.

    Added¶

    • Stationary white-noise (WN) prior for the fiscal GDP-ratios. FREV_GDP (revenue/GDP), FPEXP_GDP (primary expenditure/GDP) and SFA (stock-flow adjustment) are now estimated with a stationary white-noise prior — the own-first-lag prior mean is set to 0 (vs the default random-walk own-lag of 1) — so these mean-reverting shares revert to their in-sample means at the long end instead of drifting. Mechanism: the GLP prior in BVAR/MFBVAR/GLPreplicationWeb/bvarGLPmf.m zeroes the selected own-lag prior means (diagb(pos)=0; b(2:n+1,:)=diag(diagb)), threaded as the pos/stationary 3rd positional argument of cgl_bvarV5.m (function [...] = cgl_bvarV5(x,p,stationary,...), pos=stationary). The estimators pipeline/export_bvar_gvar24_loop.m (and the euro builder) read the per-variable prior from the spec (priorsfrom → statCols = find(Spec.Prior=="WN")) and pass it down. The effective debt rate EFFI stays RW (prior:RW, isFinancial:true). Result, e.g.: US revenue/GDP now mean-reverts to ~17.4% instead of being stuck near ~19% under the RW prior. The change is a no-op (byte-identical) for the 44 non-fiscal blocs (their statCols is empty). Verified in the deployed specs: US/EA/CA/UK/JP/CN FREV_GDP/FPEXP_GDP/SFA carry prior:WN, EFFI carries prior:RW. Committed in 6f0161a / 22250be / e38673a.
    • Out-of-sample empirical-evaluation harness (new Empirical Evaluation page; full design in the separate repo /home/farkas/github/em-bayesian-gvar-eval). A recursive expanding-window OOS forecast horse-race (proposed endogenous-US emgvar_endo + frozen-anchor emgvar_frozen vs six benchmarks: random walk + climatology, AR(p), single-country BVAR, textbook OLS GVAR, single-pass Bayesian GVAR, and a Bayesian DFM) plus a Tier-A endogenous-vs-frozen spillover contrast. The harness references the main tool via BVAR_TOOL_ROOT (it addpaths the estimator and imports gvarSolve.ts via vitest; it never copies estimator code). Origins 2010Q1–2024Q4, rolling as-of-t0 trade weights (the shipped 2021–2023 weights_full.json is forbidden in the sweep), four enforced look-ahead invariants, scoring by CRPS (primary) + parametric log-score + PIT/coverage, with Giacomini–White / Clark–West / Diebold–Mariano + Romano–Wolf + Model Confidence Set inference (matlab/oos_driver.m, eval/score.py, eval/tests.py, eval/tables.py). The empirical results are forthcoming/preliminary; the design is final.
    • Per-economy variable table in the methodology manual (docs/user_manual.tex, §Economy Coverage): every variable of every economy tabulated by transformation (log / lin) · prior (RW / WN) · financial tag, alongside new manual sections (the stationary-prior derivation, the EM-convergence and spectral-radius diagnostics, the empirical-results scaffold) and additional TikZ figures (architecture, Kalman, GVAR coupling, EM-trace schematic). Committed in fa104cd / 34d3e15.

    Changed¶

    • GLB global commons coherence — GLB_USEQ and GLB_OIL are now isFinancial:true everywhere. Both the US equity common (GLB_USEQ) and the oil common (GLB_OIL) are tagged financial across all blocs, so they are kept OBSERVED through the COVID-as-missing window (financials are not imputed) rather than NaN-masked with the macro block. Verified in the deployed EA spec (GLB_OIL/GLB_USEQ isFinancial:true, joining GLB_USLR/GLB_USBAA).
    • Canada industrial production (IP_CA) rebuilt as a log-LEVEL. The discontinued FRED level index is chained forward with the growth series to 2025Q4 and carried as a transformation:"log" log-level (pipeline/build_ca_ip_level.py), removing the level/growth incoherence in the CA bloc. Committed in 9839a46.
    • Full consistent GVAR-EM refit (33 blocs + coupling). All blocs were re-estimated together so the fiscal WN prior, the GLB financial tags, and the CA IP log-level are applied coherently across the whole system, with the coupling artifact regenerated. All 1048 per-economy variables were audited coherent (transformation · prior · financial). Committed in e38673a / 22250be.

    [2.0.0] - 2026-06-10¶

    Major release: the 24-bloc Global VAR with US-determined global commons (oil + US financial conditions), ~50-economy coverage, and a self-configuring cross-country solver. Breaking vs 1.x: the US anchor grew to n=37, every GVAR bloc carries 4 block:'global' commons, the whole panel is on a single 2025Q4 vintage (first forecast = 2026Q1), and the pre-shipped coupling artifact (coupling_default.json) is now mandatory for interactivity at 24 blocs.

    Added¶

    • Coverage expanded to ~50 economies. One registry (app/src/lib/data/countries.ts) now drives the dropdown: 6 core majors + 34 OECD members + 10 non-OECD emerging markets. Each is a single-country BVAR except the 24 that are GVAR-linked. Sources: FRED (cross-checked against national sources) for the core, Eurostat for EU/EFTA OECD, OECD-QNA / national-bank / IMF for the rest.
    • 24-bloc Global VAR (~90% of global demand). The GVAR was extended from 6 to 24 blocs: the US closed anchor + 23 Rest-of-World blocs (EA, JP, UK, CA, CN, CH, KR, MX, AU, TR, SE, PL, NO, DK, IN, BR, RU, ID, SA, AR, ZA, TH, MY). The bloc list + row-normalized trade weights live in app/public/data/bvar/gvar/weights_24bloc.json. The 23 RoW blocs are re-estimated to a cross-country fixed point by the GVAR-EM loop pipeline/export_bvar_gvar24_loop.m (each bloc's 3 stars built from partners' DK-completed series, iterated to convergence).
    • Global commons (shared US-determined variables). Each RoW bloc now also carries 4 block:'global' variables identical across blocs: GLB_OIL (WTI), GLB_USLR (US 10Y), GLB_USBAA (US BAA → credit spread), GLB_USEQ (S&P 500) — a standard GVAR global-variable treatment that transmits oil and US financial-conditions shocks to every bloc.
    • Self-configuring GVAR topology. configureGVARTopology/deriveBlocIdx (gvarSolve.ts) derive the active bloc set, per-bloc variable indices, and renormalized weights at load from the deployed spec.json files + weights_24bloc.json — no hardcoded IDX/W. A bloc is GVAR-active iff its spec carries block:'row' vars and it has a weights row.
    • Closed-form cross-country solve + precomputed coupling. The scenario fixed point is solved as x = (I − M)⁻¹d. At 24 blocs the in-browser inverse is infeasible (~19 min), so a precomputed coupling (M, (I − M)⁻¹, ρ) is shipped as app/public/data/bvar/gvar/coupling_default.json (regenerated by app/scripts/precompute_gvar_coupling.mjs) and hydrated into the solver cache, giving a sub-second matrix–vector solve — the key interactivity enabler for the full system.
    • User-defined long-run anchor (PLR). Pin a variable's long-run (≈10-year) forecast to a target; the posterior draws are entropically tilted toward it (per-draw weights, no re-estimation), pulling both the baseline and scenario fans (app/src/lib/compute/longRunTilt.ts). Single-country (non-GVAR) models only.
    • Global-common transmission in the solver (applyGlobalPins, gvarSolve.ts). The 4 block:'global' commons are now propagated end-to-end: deriveBlocIdx exposes them as IDX[c].glob (id → column), and each RoW bloc's global columns are pinned to its own baseline + the US scenario DEVIATION of the matching global (matched by id). A shock to US oil / US financial conditions therefore moves the identical common delta into every RoW bloc (verified on the real 24-bloc data: a US GLB_OIL pin gives an identical oil delta across all 23 RoW blocs to ≤1e-9), while each bloc's domestic variables respond through its own estimated coefficients. The forcing enters the per-bloc d, not the RoW↔RoW coupling M, so the shipped coupling artifact is unchanged. Gauss–Seidel ≡ closed-form to ≤1e-9 with globals present.

    🔴 ASSUMPTION (load-bearing): the globals pin the US deviation on each bloc's own baseline, not the US level. Each RoW bloc estimated its own (differing) baseline path for the globals, so a level pin would inject a 17–70-unit spurious delta at zero scenario. The delta pin gives zero spillover at zero scenario and an identical common shock under conditioning. See docs/ARCHITECTURE_FOR_AI.md §5.

    Changed¶

    • US anchor re-estimated at n=37 (was n=36). The 4 global commons (GLB_OIL/GLB_USLR/GLB_USBAA/GLB_USEQ) are tagged block:'global' so the RoW blocs match them by id, and the DDEBT_GDP fiscal driver was replaced by EFFI ("Effective rate on debt, % p.a.", block:'fiscal') — the fiscal recursion now reads an estimated effective debt rate (fiscal_meta.idx.effRate) instead of the Δdebt-column shim.
    • All 50 blocs share a single 2025Q4 data vintage, so the first forecast ("nowcast") quarter is uniformly 2026Q1. app/src/lib/dates.ts now dates every bloc from the common 2025-10-01 panel end via lastHistOf() (the old per-bloc LAST_HIST_DATE map covered only 6 blocs at stale 2025Q2/Q3 vintages, leaving the other 44 with undated axes).

    Fixed¶

    • GVAR baseline ↔ engine consistency (mean AND bands regenerated). The 24 GVAR blocs' stored baseline was fully regenerated from the shipped companion so it reproduces what the app computes in-browser:
    • Means (app/scripts/regen_gvar_baselines.mjs): the deterministic all-NaN smoother on companion_mean.json, stored at full double precision. Previously they drifted 0.07–1.77 (100·log) vs the engine — a zero-scenario showed a small spurious deviation and the baseline-reproduction canary fired; now they reproduce to 0.
    • Bands q05/q16/q84/q95 (app/scripts/regen_gvar_bands.mjs): the predictive density the chart itself draws — bvarForecastDraw(paramsDraw, hist, H, addInnovation=true) over all 500 posterior draws (parameter draw + per-step innovation), with a seeded PRNG for byte-reproducibility, reduced to fractiles in both 100·log LEVEL space (baseline_levels.json) and DISPLAY space (baseline.json). Band widths match the prior export to ~MC noise; q05≤q16≤q84≤q95 and the mean sits inside the 90% band for every cell.

    [1.4.0] - 2026-06-08¶

    Changed¶

    • COVID-as-missing re-estimation of all six models. The 2020 pandemic quarters are now treated as MISSING (NaN) and imputed with the Durbin–Koopman simulation smoother inside a data-augmentation Gibbs sampler (BVAR/MFBVAR/cgl_bvarV5.m): each sweep draws the missing data, updates the Minnesota hyperparameters (lambda, mu) by Metropolis–Hastings on the completed-data marginal likelihood, then draws the VAR coefficients. This is deliberately not Lenza–Primiceri (2022) COVID volatility scaling. The COVID window self-calibrates to the 2020Q2 GDP trough. New canonical estimators: pipeline/export_bvar_gvar_covid.m (EA/JP/UK/CA/CN), pipeline/export_bvar_us_covid.m (US, n=36, sur=0 so the 2-hyperparameter Gibbs is well-defined). The old export_bvar_gvar.m / export_bvar_us_fiscal.m (bvarGLP) are superseded.
    • lambda is now unconstrained / empirical Bayes for every model — the per-country lamfix (EA/JP/UK/CA 0.10, CN 0.04) is gone; with COVID imputed, the marginal-likelihood optimum is well-behaved (estimated lambda 0.24–0.56).
    • Uncertainty bands are now PREDICTIVE (parameter + innovation + missing-data forecast-error variance), from the Gibbs predictive density (Dforecast) in the export and bvarForecastDraw(..., addInnovation=true) in the browser. The old parameter-only bands (spread of posterior mean paths) collapsed under a tight prior and badly understated uncertainty. GDP 68% bands (q/q ann., pp): US 4.2, EA 5.0, JP 7.9, UK 9.5, CA 4.9, CN 13.5. The plotted mean stays the deterministic posterior-mean-coefficient forecast (so the unconditional smoother still reproduces it — gvar-convergence canary).

    Added¶

    • "Reset all" button (Scenario panel) — clears every scenario and all cross-country GVAR conditioning back to one empty default, behind an inline confirmation. Distinct from the per-scenario Reset (active scenario, current country only).

    Fixed¶

    • EA (and all GVAR blocs) density bands were too tight — the parameter-only methodology under EA's tight prior gave a ~1.9pp GDP band; fixed by the predictive bands + COVID-as-missing re-estimation above (~5pp).
    • No uncertainty band in the "% vs base" (deviation) view for GVAR blocs — the GVAR override was replacing the computed levelDeviationStd with the GVAR engine's null; now falls back to the single-country conditional std so the deviation band renders for EA et al.
    • Regenerated the US conforekf ground truth (pipeline/_verify/gt.m) for the scenario-scale-convention canary after the US re-estimation; companion draws compressed to 6 significant figures (US 52→26 MB).

    [1.3.0] - 2026-06-07¶

    Added¶

    • Full six-country GVAR. Every non-US model is now a GVAR bloc with its own three trade-weighted Rest-of-World stars (ROWDEM/ROWINF/ROWFIN): United States (closed anchor, n=36) + Euro Area (n=13), Japan (n=13), United Kingdom (n=15), Canada (n=15), and China (n=10). Conditioning any bloc propagates to all others through the shared RoW stars, solved as the linear cross-country fixed point Δ = M·Δ + d. The Gauss-Seidel and direct solvers agree to ~5e-11 across all six blocs (per-country and multi-country-mix, gvar-convergence.test.ts); the full system has spectral radius rho(M) ~ 0.59 and converges in ~10 sweeps. Trade weights row-normalized over the six blocs (pipeline/gvar_rawdata/weights_6bloc.json); the five non-US models are estimated by pipeline/export_bvar_gvar.m.
    • China as a full GVAR bloc. China is re-estimated as a 10-variable RoW bloc (7 domestic + 3 stars) at a tighter Minnesota prior lambda=0.04 — the wider 0.10 prior let the posterior mode extrapolate China's high-growth-2000s sample (~11% nominal GDP) far above the draw distribution, leaving export/import densities off-centre; the tighter prior re-centres the bands on the unconditional forecast. (Supersedes the earlier standalone single-country CN model; export_bvar_cn.m is deprecated.)
    • Reporting-units policy (displayUnits.ts). Index / FX / equity / commodity series (FXTWM, SP500, HP, PZTEXP, GSNE, NEER_EA, NEER_UK, NEER_CA, REER_CN, EQ_CN) are now displayed as the index level (e.g. 2010 = 100) instead of q/q-annualized growth — a trade-weighted FX or equity index q/q-ann. is not an informative number, the level is. Stored as 100·log(index) and conditioned in that space; only the display is exponentiated. baseline_levels.json now carries level-space quantiles (q16/q84/q05/q95/q50) for the level uncertainty fan, generated reproducibly inside the export scripts.
    • Level-space uncertainty fans for the level-displayed series, computed from the same posterior draws as the growth bands so the two are mutually consistent.

    Changed¶

    • All six countries refreshed to 2025Q4 from official national sources (Eurostat, ONS, StatCan, e-Stat, NBS) cross-checked against FRED, then re-estimated.
    • Unemployment-rate display bands floored at 0. LR, LR_JP, LR_UK, LR_CA can no longer render an impossible negative lower band. (A persistently-low central unemployment forecast remains a model question for re-estimation, not a display fix.)
    • GVAR cross-country slice now engages for all six blocs in the chart layer, not just US/EA/JP.

    Fixed¶

    • Experimental per-draw GVAR band dropped UK/CA/CN own-pins. A leftover 3-bloc ['US','EA','JP'] loop in ChartGrid.tsx (from the pilot) meant the experimental per-draw band for UK/CA/CN was computed without their own conditioning pins, so the band sat on the baseline while the line followed the propagated path. Now iterates the canonical six-bloc list.
    • Stale 3-bloc gvar/weights.json replaced with the six-bloc matrix (provenance only — the solver weights are compiled into gvarSolve.ts).
    • Pipeline header/dimension comments corrected to per-bloc sizes; export_bvar_cn.m marked DEPRECATED (running it would overwrite the live GVAR China model).

    [1.2.0] - 2026-06-06¶

    Added¶

    • Global VAR (GVAR) cross-country linkage. The Euro Area and Japan models now carry a Rest-of-World block of three trade-weighted "star" variables (foreign demand ROWDEM, foreign inflation ROWINF, foreign policy/financial conditions ROWFIN). Conditioning the US propagates to EA and JP, with EA<->JP back-feed, solved as a linear cross-country fixed point Δ = M·Δ + d at solve time. Two solvers (Gauss-Seidel and direct/doubling) agree to ~1e-11; the US->EA->JP pilot converges in ~4 sweeps (rho(M)~0.29). The solve runs in a debounced, stale-guarded Web Worker with a convergence read-out (sweeps, rho, ms). Trade weights come from the IMF IMTS database (build_trade_weights.py); the star series are built by build_gvar_data.py; EA/JP are re-estimated by export_bvar_gvar.m. New engine: gvarSolve.ts, runGvarSolve.ts, runGvarBandSolve.ts, gvarWorker.ts, useGvarForecast.ts, gvarLoader.ts.
    • US fiscal block (stock-flow-consistent). The US BVAR estimates government current-receipts and primary-expenditure ratios as drivers; primary/overall balance and the debt-to-GDP path are derived outside the VAR by the budget-identity recursion (CBO / IMF financial-programming style). Debt is parametrized as the change in debt/GDP and accumulated (debt = d0 + cumsum(Δd)). Conditioning a driver flows through to the derived debt path. Methodology: docs/fiscal-block-methodology.md; engine: fiscalAccounting.ts.
    • Per-draw GVAR uncertainty bands. In GVAR mode the uncertainty bands re-solve the cross-country system per posterior draw, so derived (RoW and fiscal) series get a band that is correctly centred on the propagated mean path rather than the baseline. Draw subsampling uses a deterministic stratified stride (reproducible, full tail coverage).
    • Auto-updating US dataset. build_us_auto.py rebuilds the US model hands-off from public FRED series, with the effective federal funds rate (FEDFUNDS) as the policy instrument (the 2-year yield moves to the financial block). The model auto-detects its sample span (currently to 2025Q4).
    • China (CN) added as the sixth country — a real estimated BVAR from public FRED data (17 variables, 3 lags: nominal GDP, industrial production, CPI, 3M rate, exports, imports, real effective FX, Shanghai equity + a US block), replacing an earlier synthetic placeholder. Built by pipeline/build_country_data.py + pipeline/export_bvar_cn.m.
    • Opt-in experimental features — a lower-left toggle. The first experimental feature is the per-draw iterative GVAR density (re-solves the cross-country system for each posterior draw); the default band is the faster model-specific density (see Changed).
    • Dated (calendar) x-axis on all charts, with date+value hover and dated Excel export. The left-hand conditioning sliders keep the NC / 1Q / 2Q / 3Q labels.

    Changed¶

    • US model re-estimated at n=36 with the fiscal drivers and FEDFUNDS policy rate. The Minnesota tightness is auto-selected; redundant real federal-consumption (GFH, a ~0.98-correlated subcomponent of government spending) is dropped, which restores a healthy auto-lambda (~0.28) and well-behaved conditioning.
    • EA/JP RoW models estimated with a fixed Minnesota tightness lambda=0.10: the auto-lambda optimum (~0.5-0.6) was too loose for the small n=15, T~90 systems and produced oscillating unconditional baselines; the fixed moderate-tight prior yields smooth, mean-reverting forecasts.
    • Fiscal reporting charts simplified. Under the change-in-debt/GDP parametrization the effective interest rate is backed out of the identity and degenerates to approximately nominal growth, so the effective-rate, interest (debt-service), and overall-deficit charts were removed; revenue, primary expenditure, primary balance, and debt/GDP are kept.
    • Default scenario uncertainty band reworked. The default band is now the model-specific (unconditional) forecast density parallel-shifted onto the propagated scenario path — it keeps the full forecast width (it no longer collapses when only a point or two is conditioned) and centres on the displayed line. The earlier per-draw iterative cross-country re-solve (which compressed the band and was slow) is preserved behind the experimental toggle.
    • Drag and band performance. Dragging now coalesces to one update per animation frame (rAF); posterior draws load as Float64Array (faster worker transfer + indexing); a US-viewed band skips the frozen-US's irrelevant EA/JP cross-country re-solve; and an unused Cholesky per draw was removed.

    [1.0.0] - 2026-05-30¶

    Added¶

    • Multi-country BVAR conditional forecasting for 5 economies: United States (33 variables), Euro Area (22), United Kingdom (22), Japan (22), and Canada (22). Each country BVAR includes companion-form VAR coefficients, posterior mean baselines, and historical data pre-computed from FRED series.
    • Durbin-Koopman (DK) Kalman smoother for conditional forecasting, ported from MATLAB (conforekf.m / VARcf_DKcks.m / runKF_DK.m). State-space representation with companion-form transition matrices and Cholesky-based simulation draws.
    • 155 DSGE models from the Macroeconomic Model Data Base (MMB) with pre-computed impulse response functions across 9 policy rules (Taylor, GR, LWW, SW, CEE, OW08, OW13, Coenen, CMR). Models span 6 categories: Estimated US (69), Calibrated (50), Estimated Euro Area (20), Adaptive Learning (11), Estimated/Calibrated Multi-Country (10), and Estimated Other-Country (9). A total of 726 model-rule IRF files are included.
    • Three counterfactual modes for policy analysis:
    • Hold targets -- condition on the scenario paths for all user-specified variables while applying a rate adjustment via the DK smoother.
    • Let adjust -- condition only on the policy rate adjustment and allow all macro variables to adjust freely through the BVAR dynamics.
    • IRF-consistent -- use DSGE model IRFs to compute structurally consistent inflation and output gap responses to the policy shock sequence (Toeplitz convolution), then condition the BVAR on these DSGE-implied deltas.
    • Barnichon and Mesters (2023, AER) optimal policy solver. Quadratic programming approach using sufficient statistics (DSGE IRFs) to find the optimal monetary policy shock sequence minimizing a weighted loss function over inflation and output gap deviations.
    • Per-variable, per-quarter conditioning via the control panel. Users can select any BVAR variable for conditioning, set deviation paths with sliders (-3 to +3 percentage points), and toggle individual quarters on or off. Conditioning paths are expressed relative to the DK smoother's unconditional forecast.
    • Interactive draggable chart interface using Plotly.js. Inflation, GDP growth, and policy rate charts support direct drag interaction on forecast points. Other variables can be conditioned via a "condition" button on each chart panel.
    • Uncertainty bands with two modes: Quick (analytical 16th/84th percentile bands from the DK smoother covariance) and Full (Monte Carlo simulation draws from the posterior distribution of companion matrices, loaded lazily from companion_draws.json).
    • Up to 4 simultaneous scenarios with color-coded overlays (blue, red, green, purple). Each scenario maintains independent model selection, policy rule, conditioning paths, counterfactual settings, and rate adjustments.
    • Excel export and import of scenario configurations. Export produces multi-sheet .xlsx files containing settings, conditioning paths, active quarters, rate adjustments, and a variable index map. Import supports both .xlsx and legacy .json formats, with robust variable matching across spec changes.
    • Full session export from the top bar, producing a single Excel workbook with one sheet per scenario including all settings, conditioning paths, and rate adjustments.
    • Levels vs. deviations display mode toggle, and quarter-over-quarter vs. year-over-year growth rate display for applicable variables.
    • Custom matrix algebra library (matrix.ts) using flat Float64Array storage in row-major order for browser-side linear algebra, including Cholesky decomposition, matrix solve, and Toeplitz construction.
    • Docker Compose deployment configuration with a Next.js frontend container (port 3000) and a Python backend container (port 8000) with FRED API key support and IRF caching volumes.

    Known Issues¶

    • 14 DSGE models have no pre-computed IRFs (shown as having zero available rules in the catalog). These are primarily adaptive learning variants and two other models:
    • CA_ToTEM10 -- Murchison & Rennison (2006)
    • EAUS_NAWMctww -- Cogan et al. (2013)
    • NK_BGG99AL -- Bernanke et al. (1999) AL
    • NK_CDK24 -- Chan, Diz, & Kanngiesser (2024)
    • NK_CGG02AL -- Clarida et al. (2002) AL
    • NK_CGG99AL -- Clarida et al. (1999) AL
    • NK_IR04AL -- Ireland (2004) AL
    • NK_LWW03AL -- Levin et al. (2003) AL
    • NK_RW06AL -- Ravenna & Walsh (2006) AL
    • NK_RW97AL -- Rotemberg & Woodford (1997) AL
    • US_FM95AL -- Fuhrer & Moore (1995) AL
    • US_MI07AL -- Milani (2007)
    • US_SW07AL -- Slobodyan & Wouters (2012)
    • US_YR13AL -- Rychalovska (2016)
    • Docker backend not tested end-to-end. The docker-compose.yml defines frontend and backend services, but the backend (Python/MATLAB bridge for live BVAR estimation and MMB IRF computation) has not been validated in an integrated deployment.
    • China BVAR has limited data. The CN BVAR includes 23 variables with baseline and companion matrices, but lacks companion_draws.json (no full uncertainty bands), baseline_levels.json, and historical_levels.json. The underlying data does not come from FRED and may have coverage gaps.
    • IRF-consistent mode falls back silently to let-adjust when the selected DSGE model lacks IRFs for the chosen policy rule. A small warning is shown in the UI but no explicit user confirmation is required.

    Technical Notes¶

    • All BVAR computation (Kalman filtering, DK smoothing, conditional forecasting, uncertainty simulation) runs entirely client-side in the browser using TypeScript and Float64Array-backed matrix operations. No server round-trips are required after initial data loading.
    • Pre-computed data totals approximately 152 MB, served as static JSON from /public/data/. This includes BVAR posterior estimates for 6 countries, 726 DSGE model-rule IRF files, and the model/rule catalogs. The US companion_draws.json (posterior simulation draws for full uncertainty) is loaded lazily only when the user clicks Compute.
    • The unconditional DK smoother result is cached and reused across conditioning changes to avoid redundant computation. A 30ms debounce is applied to conditioning path updates.
    • Built with Next.js 14.2, React 18, Zustand 4.5 for state management, Plotly.js for interactive charts, and SheetJS (xlsx) for Excel I/O. Styled with Tailwind CSS. Tested with Vitest.
