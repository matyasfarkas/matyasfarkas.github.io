    Fiscal Block — relation to the CBO and IMF FPP frameworks¶

    The fiscal block estimates the fiscal drivers inside the BVAR and derives the reporting variables (deficit, debt service, debt-to-GDP, gross financing needs) by accounting identity. This is the same division of labour used by CBO and by IMF Financial Programming (FPP) / debt sustainability — not a coincidence; the design is built on their shared backbone.

    The shared backbone¶

    Both frameworks separate "forecast the macro + fiscal flows" from "derive debt and the balances by accounting." Neither lets a reduced-form model freely forecast the debt stock.

    • CBO runs a conditional BVAR (Minnesota/GLP shrinkage, hierarchical tightness — the same prior family this tool already uses; Conditional Forecasting With a Bayesian VAR, 2023) purely as a macro scenario generator; those paths feed a separate, simplified budget module that produces revenues, outlays, net interest, deficit, and debt. Budget accounting lives outside the BVAR.
    • IMF FPP is the four-sector (real / fiscal / monetary / external) consistency framework. The fiscal block forecasts revenue, expenditure, and financing, then imposes the government budget constraint and debt-dynamics identity to get debt, with a stock-flow-adjustment (SFA) residual closing it. The modern quarterly incarnation is the IMF QMFF (A Novel Quarterly Macroeconomic Forecasting Framework, 2026): a QPM core + FPP sectoral accounting, linkable to the IMF debt tools (DDT / DDT_GFN).

    This tool is that division of labour: the BVAR (extended to carry the fiscal flow ratios) is the scenario generator; app/src/lib/compute/fiscalAccounting.ts is the budget/accounting module. We fuse CBO's "macro BVAR" and "budget module" into one interactive tool, with the accounting module implemented as the IMF FPP/DSA debt-dynamics recursion.

    Element-by-element mapping¶

    Concept CBO IMF FPP / DSA This tool

    Snowball / (i−g) interest-vs-growth in long-run outlook Fiscal Monitor decomposition derived snowball decomposition

    Conditioning on official/policy paths current-law & alternative scenarios WEO-consistent soft conditions the tool's scenario conditioning (drag driver paths)

    The debt recursion is the IMF DSA / DDT–DDT_GFN equation — same (1+i)/(1+gⁿ) snowball, same SFA term, same primary-vs-overall-balance distinction and sign convention (primary balance positive = surplus). Both frameworks require debt service to use the effective average rate on the stock, not the policy/market rate — hence we estimate EFFI rather than reusing the 2Y.

    The debt recursion, precisely¶

    This section pins the exact arithmetic the accounting module runs, so the manual and the code never drift. The authoritative implementation is app/src/lib/compute/fiscalAccounting.ts, which exposes two recursions:

    • computeFiscalBlock (fiscalAccounting.ts:158) — the live forecast/band path (called per draw from runGvarBandSolve.ts). It reads the drivers off the level-space path and rolls the debt identity forward. It does not apply the Bohn / risk-premium feedback (pb = revenue − primexp exactly; no γ/β term).
    • runFiscalRecursion (fiscalAccounting.ts:66) — the array-level core that adds the optional active debt-feedback levers (the Bohn fiscal-reaction-function γ and the risk-premium β).

    ⚠️ runFiscalRecursion has NO live UI/production caller. As of this writing it is invoked only from the test suite (__tests__/compute/fiscalAccounting.test.ts, __tests__/compute/fiscal-backcast.test.ts) — the γ/β-lever path is test-covered only, not wired into the production forecast flow (the band/forecast path goes exclusively through computeFiscalBlock). The feedback levers are documented and validated but currently dormant.

    The roll-forward identity below is identical in both; the feedback terms (step 1's γ, step 2's β) apply only in runFiscalRecursion. Read the units block at the top of the file (fiscalAccounting.ts:7) before editing anything here.

    Inputs (the BVAR-estimated drivers)¶

    Per forecast quarter t = 0 … H−1, read off the level-space forecast path:

    • revenue / GDP and primary expenditure / GDP — % of GDP, expressed as an annual ratio (the 4-quarter-sum over 4-quarter-nominal-GDP ratio the BVAR carries as a lin variable).
    • the effective rate EFFI (idx.effRate, % p.a., the average rate on the outstanding stock): the deployed form for all six fiscal blocs (US, EA, JP, UK, CA, CN), where interest = EFFI · d₋₁ / 100. The recursion also supports an estimated debt-service / GDP column (idx.debtService, % of GDP) — when present that is the endogenous interest and the effective rate is backed out of it — but idx.debtService is not in the deployed fiscal_meta.json of any bloc, so the live path takes the effective-rate branch.
    • stock-flow adjustment SFA (idx.sfa, % of GDP annual ratio) — read per-quarter off the level path when an SFA column is estimated, so the debt projection carries the deficit–debt discrepancy forward with its own dynamics and bands; otherwise a constant sfaDefault (or zeros).
    • nominal GDP growth gⁿ — built from the level path as gNomQ = Δ(real-GDP 100·log) + Δ(deflator 100·log) (% per quarter), i.e. ((levelPaths[t][idx.gdp] − prev[idx.gdp]) + (levelPaths[t][idx.deflator] − prev[idx.deflator]))/100 (fiscalAccounting.ts:169). The growth and rate are quarterized internally while the flow reporting variables stay annual.

    The seed d₀ (initial debt/GDP, %) and the feedback anchor d* (default d₀) come from fiscal_meta.json (d0).

    Per-quarter recursion (using lagged debt d₋₁ so the loop stays explicit)¶

    With dₚᵣₑᵥ = d₀ at t=0 and dₚᵣₑᵥ = debt[t−1] thereafter (fiscalAccounting.ts:170):

    1. Primary balance pb_t = rev_t − primexp_t (live path, fiscalAccounting.ts:171), plus the optional Bohn fiscal-reaction-function + γ·(d₋₁ − d*) in runFiscalRecursion (fiscalAccounting.ts:98; γ default 0.03 ⇒ consolidation as debt rises above d*).
    2. Interest / debt service (% of GDP): int_t = EFFI_t · d₋₁ / 100 (deployed effective-rate branch, fiscalAccounting.ts:177), or int_t = dsvc_t directly when a debt-service column is estimated; runFiscalRecursion additionally adds the debt-level risk premium β·max(0, d₋₁ − d*)·d₋₁/100 (fiscalAccounting.ts:94–95, β default 0). The effective rate is backed out for the roll-forward and for display: i_eff_t = int_t · 100 / d₋₁ (fiscalAccounting.ts:178, 96).
    3. Roll the stock forward (the IMF DSA / FPP debt-dynamics identity), quarterly: d_t = d₋₁ · (1 + i_eff_t/400) / (1 + gⁿ_t/100) − pb_t/4 + sfa_t/4  (fiscalAccounting.ts:185). Here i_eff_t/400 is the quarterized effective rate (so d₋₁·(1+iQ) = d₋₁ + int_t/4), the /(1+gⁿ/100) term is the denominator that shrinks the ratio when nominal GDP grows, and the primary balance and SFA enter as quarter-shares (/4, since they are annual ratios).
    4. Derived reporting variables, all falling straight out of the same identity:
    5. overall deficit deficit_t = int_t − pb_t (fiscalAccounting.ts:188);
    6. implied change in the ratio Δ(debt/GDP)_t = d_t − d₋₁ (fiscalAccounting.ts:186, the chart's third-row derived variable);
    7. the snowball (automatic debt dynamics) snowball_t = [(1+iQ)/(1+gQ) − 1]·d₋₁ — the interest-minus-growth (i−g) contribution the IMF Fiscal Monitor decomposition isolates (fiscalAccounting.ts:190);
    8. gross financing needs gfn_t = deficit_t + amortization_t (fiscalAccounting.ts:191).

    SFA closure and the inverse map¶

    The SFA is the soft-identity residual that closes the deficit-vs-debt gap. It is estimated as its own BVAR column (the idx.sfa path) and forecast with its own dynamics and bands; the deployed debt roll-forward, however, uses the structural constant sfaDefault (the long-run SFA mean), not the per-quarter forecast SFA column (computeFiscalBlock lines 192–200). The forecast column is dragged to spurious extremes by the δc long-run/consensus anchors' cross-response (e.g. UK −3.2→−8.2), which would collapse the projected debt path; the structural constant is the economically-correct long-run residual. The recursion is also invertible: requiredPrimaryBalance (fiscalAccounting.ts:217) solves the same equation for the primary balance that pins debt to a target path, pb_t = 4·[(d₋₁ + int_t/4)/(1 + gⁿ_t/100) − d_target,t] + sfa_t, which the tool feeds back as a conditioning constraint (revenue − primexp). Because effective rate and growth respond to the primary balance through the BVAR, the caller iterates invert → condition → re-read i,g → re-invert (1–2 passes converge).

    The exact quarterly convention here is pinned by the historical back-cast test (reconstructed debt/GDP vs observed; fixture app/src/__tests__/fixtures/us_fiscal_backcast.json) — the identity error is ≈ 0 except for the modelled SFA, exactly the consistency check both CBO and FPP demand.

    Where we follow faithfully¶

    • The two-layer separation (BVAR/flows + identity recursion) — the core of both frameworks.
    • Conditional forecasting itself: the tool's existing scenario engine (condition on a few driver paths → consistent forecast of the rest) is CBO's conditional-BVAR method and FPP's "official paths as soft information." The fiscal block just adds the accounting layer on top.
    • SFA soft-identity closure, primary/overall balance definitions, effective-rate debt service.

    Where we extend beyond the simplified versions¶

    • Endogenous, jointly-estimated fiscal flows. CBO's budget module is largely elasticity-based; we let the BVAR's reduced form capture revenue/expenditure cyclicality jointly with the macro block — closer to the ECB mixed-frequency fiscal BVAR and the IMF QMFF than to CBO's simplest module.
    • Debt-feedback channels (documented; DORMANT — not in the live path) — a risk-premium channel i_eff += β·(d₋₁−d*) and a Bohn fiscal-reaction-function pb += γ·(d₋₁−d*) (γ≈0.03 default) are implemented only in runFiscalRecursion, which has zero production callers (test-suite only, see the ⚠️ note above). The deployed computeFiscalBlock path — the one wired to every displayed debt chart and band — runs with β=γ=0 and there is no UI control for them. So the live tool has no endogenous debt→primary-balance reaction (Bohn) and no debt→yield risk premium; the debt path is a passive accounting recursion. CBO's simplified debt-service tool likewise does not endogenize macro feedback; FPP/DSA usually handle it via scenarios. These channels are a possible future extension toward IMF SRDSF "snowball realism" and the Bohn / fiscal-fatigue sustainability literature; because debt is not a BVAR variable, î_eff/p̂b are debt-agnostic baselines, so the +β/+γ terms would layer on with no double-counting (and, using lagged debt, the recursion stays explicit) were they ever activated.

    Where we deliberately simplify¶

    • No line-item decomposition — CBO splits revenue (individual income / payroll / corporate / excise) and outlays (Social Security / Medicare / Medicaid / discretionary). We estimate aggregate general-government revenue and primary expenditure; the BVAR subsumes the elasticities but not the components.
    • Not the full four-sector FPP — only the fiscal block plus its macro linkages through the BVAR's correlations (no monetary-survey or BoP identities). A fiscal-block extension of a macro BVAR, not a complete financial program.
    • SFA is a closure/scenario term, not a modelled valuation/FX/contingent-liability block (Phase 2/3).
    • No amortization/maturity profile for GFN, and no accrual↔cash bridge / mixed-frequency in v1.

    Known issues and limitations (flagged, not yet fixed)¶

    These are documented honestly because the tool is a live research instrument. They are flagged for fix, not resolved — read this section before relying on the affected output.

    1. China debt: nominal-GDP inflation double-count (material)¶

    The debt-snowball roll-forward shrinks the debt ratio by nominal GDP growth, built as gNomQ = Δ(real-GDP) + Δ(deflator) in 100·log space (fiscalAccounting.ts:169). For the five advanced fiscal blocs idx.gdp correctly points at a real GDP series, so — unlike China below — the growth term does not double-count prices. The deflator term, however, is a true GDP deflator only for the US (idx.gdp = GDPH, idx.deflator = JGDP, idx 7). The other four advanced blocs deflate with a consumer price index, not a GDP deflator (verified against each bloc's spec.json + fiscal_meta.json): EA GDP_EA real + HICP_EA (idx 6), JP GDP_JP real + CPI_JP (idx 7), UK GDP_UK real + CPI_UK (idx 8), CA GDP_CA real + CPI_CA (idx 8). Because real GDP already strips out prices, adding a CPI (rather than the matching GDP deflator) is a CPI-vs-deflator wedge in the nominal-growth term for EA/JP/UK/CA, not the double-price-counting error CN suffers.

    For China this was previously wrong but is now fixed (surgical CN-only re-estimate, 2026-06). The CN model historically shipped a single nominal GDP series (spec.json name "CN Nominal GDP (quarterly)"), so the growth term became gNomQ = Δ(nominal GDP) + Δ(CPI) and added the price level twice — biasing the CN debt-to-GDP path. The CN block has since been re-estimated against real GDP: spec.json var 0 is now "CN Real GDP (quarterly, chain-linked SA volume index, base 2011Q1=100)", CN/fiscal_meta.json sets idx.gdpIsNominal = false, and the growth term is now the correct real-GDP + deflator nominal growth used by the advanced blocs. This was a CN-only graft — the surgical fix re-estimated and replaced only the CN bloc; the other 32 GVAR blocs are byte-identical to the prior working model (the earlier full 33-bloc re-run degraded them and was reverted). The CN debt-to-GDP path is now quantitative on the same footing as US/EA/JP/UK/CA.

    2. Positional fiscal indices can go stale (fiscal_meta.json)¶

    computeFiscalBlock resolves every driver by integer column index from fiscal_meta.json (idx.gdp / deflator / revenue / primaryExp / effRate / sfa). Those indices are positional, and the MATLAB porters (export_bvar_country.m, export_bvar_gvar*.m, export_bvar_euro_fsap.m) do not regenerate fiscal_meta.json — it is maintained by separate scripts (build_fiscal_*.py, pipeline/fix_fiscal_meta_indices.mjs). So any change to a bloc's variable order silently desynchronizes the fiscal indices, and the recursion then reads the wrong columns. This was the cause of a recent EA/UK debt explosion (fixed in commit a42a2db, "Fix explosive fiscal debt (stale fiscal_meta indices)"). After any re-estimation that touches variable order, regenerate / re-verify fiscal_meta.json for every fiscal bloc before trusting the debt path.

    3. Long-run anchor tilt does not filter non-finite draws¶

    The entropic long-run-anchor tilt (app/src/lib/compute/longRunTilt.ts) can silently empty the anchor band. computeLongRunMoments writes one moment per posterior draw with no finiteness check; a singular / near-unit-root draw returns a NaN steady state (longRunTilt.ts:145), and a single NaN moment poisons the soft-max weight sum in solveEntropicTilt → all weights become NaN → the weighted-quantile band collapses to empty with no warning. The fix is to drop non-finite draws before the dual solve. (This is the tilt machinery the fiscal block can be conditioned through, hence its relevance here.)

    4. Conditional band uses the filtered (not smoothed) covariance¶

    The conditional-forecast mean is the Durbin–Koopman smoothed path, but the band variance is read from the filtered covariance filteredP[t] (app/src/lib/compute/kalman.ts:404–407). The filtered covariance at quarter t does not yet incorporate information from a future pin, so for quarters preceding a forward conditioning point the band is overstated (wider than the true smoothed band). This applies to any conditioned fiscal driver pinned at a future date.

    For the empirical out-of-sample density caveats (the GVAR predictive intervals are over-confident at short horizons — a separate, opposite issue from the conditioning band above), see the Empirical Evaluation page.

    Placement in one line¶

    A compact, interactive realization of CBO's "conditional-BVAR → budget-accounting" division of labour, with the accounting layer implemented as the IMF FPP/DSA debt-dynamics recursion (snowball + SFA). The displayed debt path is a passive accounting recursion; the active-debt-feedback and Bohn-reaction-function channels are documented and validated but dormant (test-only, no live caller) — a possible future extension. It sits methodologically between CBO's simplified budget module and the IMF's QMFF / DDT debt tools. The Step-1 historical back-cast (reconstructed debt/GDP vs observed, identity error ≈ 0 except modelled SFA) is exactly the consistency check both frameworks demand.
