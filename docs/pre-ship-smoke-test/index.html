    Pre-ship smoke test — default δc long-run anchors (CONSOLIDATED, all economies)¶

    What this is. The earlier docs/live-smell-test.md found the LIVE MFM (BVAR Scenario Tool, branch improve-live) is systemically implausible out of the box: the RW-prior baselines have a constant row that is a pure drift term, so the model has no steady state — the H=20 (5-year) terminal forecast is just "wherever the drift carried the level path." That produced inflation not anchoring (~49% of cases), negative terminal real rates (~41%), and a falling debt-snowball (~59%).

    This file does the per-country × per-variable version and asks the operational question: does a single, simple, publicly-known, common-sense long-run anchor (one δc per variable) make the forecast sensible? If yes, that anchor is a recommended ship-with default the tool can present as a one-click preset. Method is the verbatim app/src/lib/compute/driftSetter.ts algebra (companion T from betahat, S_h=Σ_{k<h}Tᵏ, sens_j=4·(S_H−S_{H-1})[j,j] for log / S_H[j,j] for lin, δc_j=(anchor−terminal)/sens_j; glided terminal = anchor by construction). Replicated and the identity holds. READ-ONLY — no app/src or data files were modified.

    The consolidated synthesis below covers all 46 graded blocs; the per-alphabet-band sections that follow (D–G, H–K, L–N, S–Z, A–C, plus PL/PT/RU folded in) are the full supporting detail.

    4. Ship verdict¶

    YES — presentable, with the default anchors shipped as one-click presets and four labels.

    • 96.5% (219/227) of every smell across 46 blocs is resolved by a single, publicly-defensible δc. With the default-anchor preset applied, the out-of-the-box forecasts anchor at each central bank's inflation target, a positive neutral real rate, IMF-WEO potential growth, and the country NAIRU — i.e. sensible by inspection. The δc overlay is a band-invariant rigid translation, so shipped uncertainty is unchanged.
    • The 8 non-resolving cases are not bugs — they are known categories that need a different lever and a UI label, not a fix: UK & SK debt direction (primary-balance identities), the BE/CA/CZ perverse policy-rate sliders (→ disable + label the near-singular slider), CN deflation (structural reflation), AR nominal block (hyperinflation regime). All five economies still have sensible defaults for their other variables.
    • Four ill-conditioned-but-finite sliders to flag, not auto-ship the raw δc: US POLRATE (sens 0.51 → anchor π/g, let 10Y/EFFI carry the real rate), JP POLRATE (sens 0.53), SK EFFI (sens 0.56), PT LENDR (sens 0.52). PL POLRATE (sens −9.94) works but has a counter-intuitive sign → tooltip.

    Recommendation: ship the §2 preset as the default slider positions (replacing the degenerate RW drift), with UI labels on the 8 structural/regime exceptions and the 4–5 sensitive sliders. The tool is presentable out of the box. The remaining structural items (UK/SK debt direction, AR/CN regimes) are the trend-cycle / fiscal-rebuild track, not a blocker for the anchored-default ship.

    Scope. The LIVE MFM (BVAR Scenario Tool, branch improve-live) ships RW-prior BVAR baselines whose constant row is a pure drift term, so the model has no steady state — the H=20 (5y) terminal forecast is just "wherever the drift has carried the level path."     Economies graded: DE, DK, ES, FI, FR, GR. (All European; DK is non-euro but pegs the krone to the euro under ERM-II, so it imports the ECB's de-facto 2% objective and runs its own policy rate a hair above the ECB to defend the peg.)

    Method (verbatim app/src/lib/compute/driftSetter.ts). Companion T is built from companion_mean.json betahat (row 0 = constant dropped; T[i,j]=betahat[j+1][i]; identity shift in the lower block). S_h = Σ_{k<h} Tᵏ. Terminal display forecast is read off baseline.json mean[H-1]. Sensitivity of variable j's terminal display to a unit δc on its own equation: - log vars (GDP, HICP → chart shows 4·Δlevel = q/q annualised %): sens_j = 4·(S_H−S_{H-1})[j,j] - lin vars (rates, ratios → chart shows the level): sens_j = S_H[j,j]

    Required anchor shift: δc_j = (anchor − terminal) / sens_j. The glided terminal is, by construction, exactly the anchor. (Replicated in /tmp against the TS; identity holds.)

    Anchor basis (all PUBLIC reference values — not tuned to the model): - Inflation → the central bank's stated target. ECB 2.0% (DE, ES, FI, FR, GR). Denmark: ERM-II krone peg ⇒ imports the ECB's 2.0% objective. - Real growth → IMF WEO / OECD potential-growth consensus: DE ~0.8, DK ~1.5, ES ~1.5, FI ~1.2, FR ~1.3, GR ~1.0. - Policy rate → neutral = r* + π*. Euro area ECB neutral ≈ 2.0–2.5% (we use 2.25). DK ≈ ECB neutral (peg). - Unemployment → structural / NAIRU (OECD/EC estimates): DE ~3.5, DK ~4.5, ES ~11.5 (persistently high), FI ~7.0, FR ~7.0, GR ~11.0.

    resolves = true ⇔ the anchor is defensible AND the required δc is a modest constant-shift (the model is merely drifting, not structurally broken). It would be false only if a long-run shift cannot fix it (e.g. UK debt = a primary-deficit identity, not a snowball drift — not present here).
